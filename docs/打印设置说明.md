# 报价表打印设置说明（README）

> 本文档梳理 8 种案件（韩语×3、日语×3、英语×2）共 24 个工作表的**打印/页面设置**现状，
> 并分析「打印设置前端化、按业务员个人习惯自定义」的可行性。

---

## 一、打印设置字段说明

每张报价表 sheet 生成完成后，会写入以下 `openpyxl` 页面属性：

| 字段 | 取值 | 含义 |
|------|------|------|
| `page_setup.paperSize` | `9` | A4 纸张 |
| `page_setup.orientation` | `portrait` / `landscape` | 纵向 / 横向 |
| `page_setup.fitToPage` | `True` | 开启「缩放到合适大小」 |
| `page_setup.fitToWidth` | `1` | **宽度缩放到 1 页**（所有列一页） |
| `page_setup.fitToHeight` | `0` | 高度不限（0 = 自动分页） |
| `print_options.horizontalCentered` | `True/False` | 打印时水平居中 |
| `page_margins` | `PageMargins(...)` | 上下左右页边距（英寸） |

### 「一页」两种模式的区别

| 模式 | fitToWidth | fitToHeight | 打印效果 |
|------|------------|-------------|----------|
| **所有列设为一页**（当前所有案件） | `1` | `0` | 宽度强制 1 页，高度不限→行多时纵向自然翻页 |
| **全部设为一页** | `1` | `1` | 宽高都缩到 1 页，内容多时字会被压得很小 |

> 现状：**全部 24 个 sheet 都用「所有列一页宽 + 高度不限」，没有任何 sheet 用「全部一页」**。

## 三、关键差异小结

- **横向打印**：仅 `ap_common`（亚太分销）一张。
- **水平居中** (`horizontalCentered=True`)：只有日语 `ja_normal` 全部 + `ja_nv` 的围栏明细开启；其余都不居中。
- **页边距三档**：
  - `0.75/0.7` — 韩语组、ja_EST、en_simple、ap_common（最常见，最宽）
  - `0.5/0.25` — en_common（窄边距，容纳卖家/买家完整信息）
  - `1.2cm/0.4cm/1.2cm` — ja_normal 与 ja_nv 围栏（接近 1.2cm）
- **缩放**：统一为「所有列一页宽 + 高度不限」，无「全部一页」。

## 四、前端可配置性分析（思考，未改代码）

> ✅ **已落地实现**（见下方第四节「实现说明」）。本节保留原始设计分析。

### 4.1 可行性结论：**完全可行，且基础设施已就绪**

系统已有成熟的「**按用户存储个人习惯**」能力，可直接复用：

| 现有能力 | 位置 | 作用 |
|----------|------|------|
| `accounts` 表 `preferences` 字段（JSON） | user_repository.py:67 | 每个账号一个 JSON 配置 |
| `update_user_preferences()` 深度合并 | user_repository.py:553 | 增量保存，不覆盖其它键 |
| `POST /api/auth/me/preferences` | auth.py:41 | 前端保存习惯的现成接口 |
| 登录返回 `account.preferences` | auth_service.py | 生成报价时可直接读取 |

> 即「业务员个人习惯」的存储、合并、读取、鉴权链路都已存在，无需新建表。

### 4.2 推荐的配置粒度

打印设置本质只有 **3~4 个开关**，建议做成「**全局偏好 + 可选按案件覆盖**」两层：

```
account.preferences = {
  "print": {
    "default": {                       // 全局默认习惯
      "orientation": "portrait",       // portrait / landscape / auto
      "fit_mode": "fit_width",         // fit_width(所有列一页) / fit_one(全部一页) / none
      "horizontal_centered": false,
      "margin_top": 0.75,
      "margin_bottom": 0.75,
      "margin_left": 0.7,
      "margin_right": 0.7
    },
    "overrides": {                     // 可选：某案件单独覆盖（不填则用 default）
      "ja_normal_detail": { "horizontal_centered": true },
      "ap_common": { "orientation": "landscape" }
    }
  }
}
```

- 键名建议用「`案件名` + `_` + `sheet类型`」（如 `ko_normal_detail`、`en_common_summary`），
  与本文件第二节的案件命名一致，便于映射。
- 前端在「个人中心/习惯设置」页用下拉、单选、数字框呈现这 4~5 个字段即可，无需复杂 UI。

### 4.3 后端落地思路（评估，未实施）

1. **读取**：`generate` 接口里从当前登录账号 `account['preferences']['print']` 取配置，
   传给各案件的引擎函数。
2. **默认回退**：用户未设置时，沿用现有硬编码默认值（即本文第二节的现状），保证向后兼容、零回归。
3. **应用点**：每个引擎里那段 `ws.page_setup.* / ws.page_margins = ...`，
   改为「`用户值 or 现有默认`」。由于这些值都集中在每个函数末尾，改造点清晰、风险低。
4. **约束**：`orientation` 仅 `auto` 时才允许跟随内容自动判断（一般固定 portrait/landscape）。

### 4.4 设计取舍建议

| 选项 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **A. 仅全局打印偏好** | UI 简单（5 个字段），改动最小 | 无法对某案件单独调 | ★ 推荐 MVP |
| **B. 全局 + 按案件覆盖** | 灵活，老业务员可精细定制 | UI 略复杂、配置量多 | 后续可扩展 |
| **C. 每个 sheet 单独存** | 最灵活 | 配置爆炸，难维护 | 不建议 |

> 建议先做 **A**：一个业务员通常只有一种打印习惯（比如「汇总表全居中、纵向、边距窄一点」），
> 覆盖 90% 需求；少数人需要按案件微调时，再升级到 B。

### 4.5 需要确认的点（实施前）

1. 哪些案件/角色允许改打印设置？（如只放开韩语组/日语组，admin 全开）
2. 边距输入用**英寸**还是**厘米**？现状混用（英语英寸、日语接近厘米），建议前端统一显示 cm 并换算。
3. 是否需要「**恢复默认**」按钮 → 清空 `preferences.print` 键。
4. ap_common 横向是否允许业务员改纵向？（可能影响排版）

## 六、实现说明（已上线）

### 6.1 后端

* **新增 `backend/core/print_settings.py`**
  * `CASE_DEFAULTS`：9 个案件的默认打印参数（提取自原硬编码值，单位统一为**英寸**）。
  * `SHEET_DEFAULTS`：案件内个别 sheet 覆盖（`en_common_total_materials`、`ja_nv_fence`）。
  * `resolve_print_settings(case_key, sheet_key)`：默认值（sheet > case > 全局回退）
    与「当前登录用户的个人习惯覆盖（按 case）」合并。
  * `apply_print_setup(ws, case_key, sheet_key=None)`：把最终参数写回 worksheet。
* **24 处硬编码 `ws.page_setup.*` 全部替换**为 `apply_print_setup(ws, ...)` 调用，
  覆盖 ko_normal / ko_simple / ko_ksd / ja_EST / ja_normal / ja_nv / en_simple / en_common / ap_common / ja(旧)。
* **`_deep_merge` 增强**（user_repository.py）：合并时遇到 `None` 值则**删除该键**，
  用于实现「恢复默认」。

### 6.2 个人习惯存储

复用既有体系，**未新建表**：

```
accounts.preferences.print[<case_key>] = {
  "orientation": "portrait" | "landscape",
  "fit_mode": "fit_width" | "fit_one",
  "horizontal_centered": true | false,
  "margin_top": 0.75, "margin_bottom": 0.75,    # 英寸
  "margin_left": 0.7,  "margin_right": 0.7
}
```

* 存/取走现有 `POST /api/auth/me/preferences` + `update_user_preferences`。
* 用户未设置时自动回退默认值（零回归）。
* 读取发生在请求上下文内（生成报价时），非请求上下文（CLI/离线）回退默认。

### 6.3 前端（`frontend/js/pages/quotation.js`）

* 新增「🖨️ 打印设置（个人习惯）」卡片：方向 / 缩放 / 水平居中 / 四向边距（英寸）。
  卡片顶部显示「当前案件」标签，**随案件类型切换自动刷新**（ko/ja/en 单选联动）。
* **保存**：并入「保存我的习惯」按钮，与其它报价习惯一起提交。
  当输入值与默认一致时发送 `{print:{}}`（不写入），保持偏好干净。
* **恢复默认**：独立按钮，发送 `{print:{<case_key>: null}}` 删除该案件自定义并回填默认值。
* `ap_common` 默认横向，业务员可改为纵向（按要求放开）。

### 6.4 验证

* 12 个引擎文件 `py_compile` 通过；`quotation.js` `node --check` 通过。
* 默认解析与原硬编码值完全一致（如 ja_normal top=1.2/2.54=0.4724、bottom=0.4/2.54=0.1575）。
* E2E：写入 admin 偏好 → 生成时读取生效 → `null` 恢复默认 → 键被删除，回退 landscape。
