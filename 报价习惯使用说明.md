# 报价习惯（我的偏好）使用说明

每位业务员的报价习惯不同。本功能让每个人可以**把自己常用的报价参数保存为"我的习惯"**，下次打开报价页时自动回填，无需每次重复设置。

---

## 一、怎么用

### 保存习惯
1. 打开报价页，按自己的习惯填写各项参数（贸易方式、折扣、关税、消费税等）
2. 点击页面底部工具栏的 **「保存我的习惯」** 按钮
3. 提示「习惯已保存」即成功，数据写入数据库 `ks_users` 表的个人账号下

### 自动回填
- 每次进入报价页，系统会**自动读取**你的习惯并填入对应输入框
- 日语组切换案例（EST / NORMAL / NV）时，会自动切换到对应案例的习惯

### 优先级
```
输入框初始值  =  我保存的「个人习惯」  →  没有则用系统默认值
```
- 你**只保存过哪几项，哪几项就生效**；没保存过的项仍走系统默认，不会变成空白
- 报价生成时，始终以**输入框当前的值**为准（保存习惯只改变"初始默认值"）

### 注意
- 习惯是**按账号**保存的，换电脑登录同一账号也能用
- 管理员编辑你的账号信息时，**不会清空**你的习惯
- 习惯**分语言组**独立保存，各组互不影响

## 二、哪些参数可以保留习惯

下面列出**每个组可以保存的习惯字段**（共三组）。✅ 表示可以保留，⚪ 表示每次报价都会变、**不保留**。

### 韩语组

| 习惯字段 | 含义 | 对应界面 | 系统默认 | 可保留 |
|---|---|---|---|---|
| trade_method | 贸易方式（EXW / FOB / CIF / DDP） | 贸易方式单选 | CIF | ✅ |
| sale_type | 内贸 / 外贸 | 内贸外贸单选 | 外贸(export) | ✅ |
| dest_port | 目的港 | 目的港下拉 | 부산 | ✅ |
| ko_company_discount | 公司折扣（铝折扣组件①） | 公司折扣输入框 | — | ✅ |
| ko_commission | 佣金（铝折扣组件②） | 佣金输入框 | — | ✅ |
| ko_steel_discount_rate | 碳钢折扣 | 碳钢折扣输入框 | 84 | ✅ |
| ko_purchased_discount_rate | 采购件折扣 | 采购件折扣输入框 | 94 | ✅ |
| ko_tariff_rate | 关税 | 关税输入框 | 1.6 | ✅ |
| ko_consumption_tax | 消费税 | 消费税输入框 | 10 | ✅ |
| coating_thickness | 涂层厚度（10 / 15 / 18um） | 涂层厚度按钮 | 10um | ✅ |
| container_type | 柜型 / 数量 | 柜型勾选 | 40HQ×1 | ⚪ 不保留 |
| ko_freight | 海运费 | 海运费输入框 | 0 | ⚪ 不保留 |
| ko_cif_freight | CIF 运费 | CIF运费输入框 | 0 | ⚪ 不保留 |
| ko_ddp_address | DDP 地址 | DDP地址输入框 | 空 | ⚪ 不保留 |

> 💡 **铝折扣** = 公司折扣 + 佣金（界面上的铝折扣是自动计算结果，不可直接填）。保存时会自动记住这两个组件，回填后重新计算。

## 三、为什么不保留柜型 / 运费 / 海运费 / 备品？

这些字段**每次报价几乎都不一样**（取决于具体订单的出货方式、货量）。如果保留它们，下次打开会带上一次的临时数据，反而容易出错，所以设计上**排除在外**。这些字段每次都从系统默认值开始，由业务员当次填写。

## 四、技术说明（给开发/维护）

- **存储位置**：`ks_users` 表的 `preferences` 列（JSON 文本），按账号隔离
- **JSON 结构**：按语言组嵌套，日语组再按案例嵌套，避免同名字段跨组冲突
  ```json
  {
    "韩语组": { "trade_method": "CIF", "ko_company_discount": 75, ... },
    "日语组": {
      "EST": { "ja_tariff_rate": 1.6, ... },
      "NV": { "nv_company_discount": 70, ... }
    },
    "英语组": { "en_trade_method": "FOB", ... }
  }
  ```
- **接口**：
  - 读取：`GET /api/auth/me`（登录后自动随账号返回）
  - 保存：`POST /api/auth/me/preferences`（业务员自助，无需管理员权限；后端深度合并，不覆盖其它组）
- **前端逻辑**：`frontend/js/pages/quotation.js` 中的 `PREF_FIELD_MAP`（字段映射）、`applyUserPreferences`（自动回填）、`saveMyPreferences`（保存）
- **数据库迁移**：自动执行 `ALTER TABLE ks_users ADD COLUMN preferences`，老用户得到空习惯，行为不变
